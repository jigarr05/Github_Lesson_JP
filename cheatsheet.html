Git & GitHub Cheatsheet

A start-to-finish walkthrough of the commands you'll actually use — from installing Git to opening your first pull request and beyond.

01

Setup & Configuration

Run once after installing Git, so every commit is signed with your identity.

CommandDescription
git --versionCheck that Git is installed and see its version
git config --global user.name "Name"Set the name attached to your commits
git config --global user.email "you@example.com"Set the email attached to your commits
git config --global init.defaultBranch mainMake new repos default to a branch named main
git config --listShow all current configuration values
git config --global core.editor "code --wait"Set your default editor (example: VS Code)Swap in vim, nano, subl, etc. as preferred
02

Starting a Project

Every project either begins fresh locally or is copied down from GitHub.

CommandDescription
git initTurn the current folder into a new Git repository
git clone <url>Copy an existing remote repository (e.g. from GitHub) to your machine
git clone <url> <folder-name>Clone into a folder with a custom name
git statusShow changed, staged, and untracked files right nowThe command you'll run more than any other
03

Staging & Committing

The core loop: edit files, stage the changes you want to save, then commit them with a message.

CommandDescription
git add <file>Stage a specific file
git add .Stage all changed files in the current directory
git add -pInteractively stage changes hunk by hunk
git commit -m "message"Save staged changes as a new commit with a message
git commit -am "message"Stage all tracked, modified files and commit in one stepSkips new (untracked) files
git commit --amendEdit the message or contents of the last commit
git diffShow unstaged changes since the last commit
git diff --stagedShow staged changes not yet committed
04

Branching & Merging

Branches let you work on features in isolation before folding them back in.

CommandDescription
git branchList local branches (current branch is starred)
git branch <name>Create a new branch
git switch <name>Switch to an existing branch
git switch -c <name>Create a branch and switch to it in one step
git checkout <name>Older equivalent of git switch
git merge <branch>Merge the named branch into your current branch
git branch -d <name>Delete a branch that's already merged
git branch -D <name>Force-delete a branch, merged or not
git branch -m <new-name>Rename the current branch
Tip: keep main always deployable — do your work on a feature branch and merge in once it's reviewed.
05

Connecting to GitHub (Remotes)

A "remote" is a pointer to a repository hosted elsewhere, typically GitHub.

CommandDescription
git remote -vList remotes and their URLs
git remote add origin <url>Link a local repo to a GitHub repo, named origin
git remote set-url origin <url>Change the URL of an existing remote
git remote remove originUnlink a remote
gh repo createCreate a new GitHub repo from the terminalRequires the GitHub CLI (gh)
06

Syncing: Push, Pull & Fetch

Moving commits between your machine and GitHub.

CommandDescription
git push -u origin mainPush and link your local branch to origin/main for the first time
git pushPush committed changes to the linked remote branch
git push origin <branch>Push a specific branch to GitHub
git fetchDownload remote changes without merging them into your work
git pullFetch and merge remote changes into your current branch
git pull --rebaseFetch and replay your local commits on top instead of merging
07

Inspecting History

Ways to look back at what's happened in the repository.

CommandDescription
git logShow commit history, newest first
git log --oneline --graph --allCompact, visual history across all branches
git show <commit>Show the details and diff of a specific commit
git blame <file>Show who last changed each line of a file
08

Undoing Things

Mistakes are normal — Git has a command for almost every kind of "oops."

CommandDescription
git restore <file>Discard unstaged changes in a file
git restore --staged <file>Unstage a file, keeping its edits
git reset --soft HEAD~1Undo the last commit, keep changes staged
git reset --mixed HEAD~1Undo the last commit, keep changes unstaged
git reset --hard HEAD~1Undo the last commit and discard its changes entirelyDestructive — no way back once run
git revert <commit>Create a new commit that undoes a previous oneSafe for commits already pushed/shared
09

Stash & Clean

Temporarily set aside work, or clear out files Git isn't tracking.

CommandDescription
git stashShelve current changes and return to a clean working directory
git stash listShow all stashed changesets
git stash popReapply the most recent stash and remove it from the list
git stash dropDelete a stash without applying it
git clean -fdRemove untracked files and directoriesUse -n first to preview what would be deleted
10

Collaboration on GitHub

The parts of the workflow that live on GitHub itself, plus the CLI shortcuts for them.

Command / ActionDescription
ForkCopy someone else's repo to your own GitHub account to work on independently
gh repo fork <owner/repo>Fork a repo from the terminal (GitHub CLI)
Pull Request (PR)A request to merge changes from one branch/fork into another, opened on GitHub.com
gh pr createOpen a pull request from your current branch
gh pr listList open pull requests on the repo
gh pr checkout <number>Check out someone else's PR locally to test it
gh pr merge <number>Merge an approved pull request
gh issue createOpen a new issue on the repo
gh issue listList open issues
CODEOWNERS / Branch protectionRepo settings that require review before merging to main
11

Advanced Git

Reach for these once the basics are comfortable.

CommandDescription
git rebase <branch>Replay your commits on top of another branch for a linear history
git rebase -i HEAD~3Interactively reorder, squash, or edit the last 3 commits
git cherry-pick <commit>Apply one specific commit from another branch onto yours
git tag v1.0.0Mark the current commit as a named release point
git push --tagsPush local tags to the remote
git bisect startBinary-search commit history to find which one introduced a bug
git submodule add <url>Embed another repo inside this one as a submodule
git reflogShow a log of everywhere HEAD has pointed — a safety net for recovering "lost" commits
git worktree add <path> <branch>Check out another branch into a separate folder simultaneously
12

.gitignore Patterns

Common syntax for telling Git which files to never track.

PatternDescription
node_modules/Ignore a folder and everything inside it
*.logIgnore every file with a given extension
.envIgnore a specific file by name
!important.logNegate a pattern — always track this file even if a rule above ignores it
/buildOnly ignore build at the repo root, not nested ones
git rm -r --cached <file>Stop tracking a file that's already committed, without deleting it locally
